🔧 Fixa Storage Bucket

❌ Problemet du har:

  • Bucket "tracks" finns ✅
  • Filer finns i bucketen ✅
  • Men bucketen är INTE markerad som public ❌
  • URL:en försöker använda /public/tracks/ men bucketen är privat
  • Därför får du 400 Bad Request

✅ Lösningen:

Gör bucketen "tracks" public. Detta gör att URL:er med /public/tracks/ fungerar.

Steg 1: Kontrollera bucket-status

Steg 2: Gör bucketen public (AUTOMATISKT)

Detta kommer att:

  • Uppdatera "tracks" bucket till public
  • Göra alla filer i bucketen åtkomliga via public URL
  • Fixa 400 Bad Request-felet

Steg 3: Testa att det fungerar

Alternativ: Via Supabase Dashboard

Om knappen ovan inte fungerar, gör detta manuellt:

  1. Gå till Supabase Dashboard
  2. Öppna Storage → tracks
  3. Klicka på Configuration (kugghjulet)
  4. Aktivera "Public bucket"
  5. Klicka Save

Eller kör denna SQL i SQL Editor:

UPDATE storage.buckets 
SET public = true 
WHERE name = 'tracks';

📋 Vad händer när bucketen är public?

  • ✅ URL:er med /public/tracks/filename.mp3 fungerar
  • ✅ Vem som helst kan lyssna på låtarna (som det ska vara)
  • ✅ Du behöver fortfarande RLS-policies för upload/delete
  • ✅ Endast autentiserade användare kan ladda upp
